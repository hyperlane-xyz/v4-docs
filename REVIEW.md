# Code Review Guidelines

## Content Quality

- Factual accuracy of protocol descriptions and code examples
- Broken or outdated internal links (relative paths without `.mdx` extension)
- Inconsistent Hyperlane terminology (ISM, Warp Route, Mailbox, etc.)
- Code snippets that reference deprecated SDK/CLI APIs
- Missing context that would leave a user unable to complete a task

## Structure

- New pages added to `docs.json` navigation
- Logical placement within existing doc hierarchy
- Consistent use of Mintlify components (Cards, Tabs, CodeGroups)

## Never flag

- Minor prose style preferences
- Formatting handled by Mintlify
- Image optimization choices
